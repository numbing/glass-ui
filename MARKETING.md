## FAQ for Marketing

**Q: How is it so much smaller?**
A: We use static CSS instead of CSS-in-JS. No 30KB runtime library,
no style computation overhead. Just clean, pre-compiled CSS.

**Q: Does smaller mean fewer features?**
A: No! We have 26+ components with full functionality. We're just
more efficient about how we deliver them.

**Q: Will this work with my existing app?**
A: Yes. It's just React components and CSS. No special setup,
no vendor lock-in. Drop it in and it works.

**Q: Is it really 40% faster?**
A: Yes, based on benchmarks rendering 1000 components. Real apps
see similar improvements. Run `npm run benchmark` to test yourself.

**Q: What about SSR/Next.js?**
A: Works perfectly. Static CSS is SSR-friendly‚Äîno hydration issues,
no critical CSS extraction needed.
